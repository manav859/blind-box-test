# Theme Block Settings

The Blind Box product shell exposes merchant-configurable settings in the theme editor.

## Current Settings

- `show_badge`
  Shows or hides the Blind Box badge row.
- `show_icon`
  Shows or hides the decorative icon treatment next to the badge.
- `badge_text`
  Controls the badge label, such as `Blind Box` or `Collectible Drop`.
- `title`
  Main product-page shell title.
- `description`
  Short explanatory copy shown under the title.
- `display_variant`
  Layout treatment for the block: `compact`, `standard`, or `highlighted`.
- `detail_level`
  Information density for the message: `compact` or `expanded`.
- `show_how_it_works`
  Shows or hides the purchase-flow explainer card.
- `show_rarity_note`
  Shows or hides the collectible note card even if note text exists.
- `rarity_note`
  Optional collectible / rarity note.
- `show_disclaimer`
  Shows or hides the store disclaimer card even if disclaimer text exists.
- `disclaimer`
  Optional disclaimer or store note.

## Display Variants

The current display variants are:

- `compact`
  Tighter spacing for dense product layouts or when the page already has a lot of surrounding merchandising.
- `standard`
  Balanced default card with full explanatory hierarchy.
- `highlighted`
  Stronger accent treatment for featured blind-box launches or hero products.

## What Merchants Can Control Safely

These settings are purely visual or copy-driven.

They should be used on blind-box product templates, not generic product templates where the messaging could confuse shoppers.

They do not control:

- assignment behavior
- prize selection
- checkout behavior
- inventory execution
- fulfillment state

That logic remains backend-owned.
