## Current `app/` Backend Structure

The backend workspace is a small embedded Express app.

Important files:

- [app/src/index.ts](C:/Users/manav/blindbox-test-01/app/src/index.ts:1)
  creates the Express server, registers auth routes, registers `/api/webhooks`, protects `/api/*`, serves the frontend, and enforces app installation status
- [app/src/shopline.ts](C:/Users/manav/blindbox-test-01/app/src/shopline.ts:1)
  creates the SHOPLINE app instance, loads env vars, configures auth, scopes, webhook registration, and session storage
- [app/src/controller/webhook/index.ts](C:/Users/manav/blindbox-test-01/app/src/controller/webhook/index.ts:1)
  authenticates incoming webhooks and switches on starter topics
- [app/src/controller/product/create.ts](C:/Users/manav/blindbox-test-01/app/src/controller/product/create.ts:1)
  sample authenticated controller that uses the merchant session access token
- [app/src/service/product/create.ts](C:/Users/manav/blindbox-test-01/app/src/service/product/create.ts:1)
  sample SHOPLINE OpenAPI call that creates a test product

Current backend flow:

1. auth starts at `shopline.config.auth.path`
2. auth callback uses `shopline.auth.callback()`
3. after auth, the app registers configured webhooks
4. `/api/webhooks` accepts raw text and verifies webhook authenticity
5. `/api/*` is protected by `shopline.validateAuthentication()`
6. the backend serves the `web` workspace frontend and confirms installation status before loading it

Current backend folders that are safe to extend:

- `app/src/controller/`
- `app/src/service/`
- new sibling folders under `app/src/` such as `repository/`, `domain/`, `integration/`, `lib/`, and `db/`

Backend parts that should not be broken:

- `shopline.cspHeaders()`
- static serving of the `web` build output
- installation confirmation middleware

## Current Session And Auth Setup

The current embedded auth/session flow is already working and must be preserved during Phase 1.

Backend auth/session details:

- `app/src/shopline.ts` uses `shoplineApp(...)`
- environment variables provide:
  `SHOPLINE_APP_KEY`, `SHOPLINE_APP_SECRET`, `SHOPLINE_APP_URL`, `SCOPES`
- `authPathPrefix` is `/api/auth`
- the app is configured as `isEmbeddedApp: true`
- session storage is `new SQLiteSessionStorage(DB_PATH)`
- after auth, configured webhooks are registered for the authenticated merchant session

Frontend auth/session details:

- `useAppBridge()` creates the embedded app bridge client
- `useAuthenticatedFetch()` gets a session token via `shared.getSessionToken(app)`
- authenticated requests send:
  `X-Requested-With` and `Authorization: Bearer <session-token>`
- if the backend signals reauthorization headers, the frontend redirects back through the auth flow

What must not be changed casually:

- the auth path prefix
- the App Bridge bootstrap pattern
- the session-token fetch behavior
- the protected `/api/*` middleware flow

## Immediate Blind Box Implication

This repo is a good scaffold for Blind Box development, but it is still missing all business-critical modules:

- blind-box domain persistence
- blind-box repositories and services
- paid-order webhook processing
